===============================================================================
ASCII SLIDE DESIGN: GitHub Copilot for everyone
===============================================================================

Source HTML : docs/brainstorms/2026-05-15-github-copilot-for-everyone-deck.html
Audience    : non-technical managers
Goal        : explain Copilot as an everyday work agent, then demo it
Style       : Micron light, editorial, restrained, purple accent
Format      : 8 slides
Date        : 2026-05-19

Core thesis:
  GitHub Copilot started as coding help.
  It now behaves more like an agent for everyday computer work.
  The important habit is asking clearly, reviewing carefully, and reusing skills.

Story spine:
  01  Why are we here?
  02  How did Copilot evolve?
  03  Who is it for now?
  04  What is the usage loop?
  05  What can I ask it to do?
  06  How do we try it live?
  07  What are agent skills?
  08  How does an agent find and use the right skill?

SLIDE 08 - PROGRESSIVE DISCOVERY
===============================================================================

HTML content:
  Section   08 - Progressive discovery
  H2        The agent discovers the right skill as it works
  Subtitle  Example: Turn this folder of notes into slides.

  Flow:
    1 User goal
      "Turn this folder of notes into slides."
      The goal is clear. The method is not.

    2 Scan skill descriptions
      Slide Drafting    -> create slide outlines from notes
      Meeting Summary   -> turn transcripts into action items
      Data Analysis     -> analyze CSVs and explain trends
      Email Writing     -> draft messages from context
      Browser Task      -> complete a web workflow
      Caption: Scan first. Open what matters.

    3 Open the selected skill
      Slide Drafting
      Steps, Rules, Tools, Examples, Output format
      Caption: Now read the full details of the chosen skill.

    4 Apply + output
      Slide draft ready
      The agent follows the selected skill to produce the result.

  Bottom:
    Progressive discovery keeps the agent efficient: scan first, open what
    matters, then act.
