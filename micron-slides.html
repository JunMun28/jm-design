01 / Mental model

Codex is a coding agent for your repo.

01

Understand

Reads files, follows project context, and explains what it found.

02

Change

Edits code, docs, tests, configs, and scripts inside the repo.

03

Verify

Runs checks, reports failures, and shows what still needs review.

Treat it like a teammate with terminal access.

04 / Use it well

Give Codex small, reviewable work.

scope One task at a time

Ask for a file explanation, one bug fix, or one small UI change.

intent Say what good means

Tell it constraints, target behavior, and what to avoid.

verify Never skip the check

Inspect the diff, run the relevant command, then iterate.
