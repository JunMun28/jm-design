# AI for Lesson Planning — Brainstorm
**Audience:** Public school teachers (K-12, mixed subjects, mixed tech comfort)
**Length:** ~6 slides / ~15-20 min talk
**Tone goal:** Warm, trustworthy, peer-to-peer. NOT "look how cool the tech is."

---

## Assumptions I'm making (since I can't ask)

- Mixed grade levels and subject areas in the room.
- Most teachers have *heard* of ChatGPT, a few have tried it, some are wary.
- The wariness is real and reasonable: cheating, privacy, accuracy, "is this going to replace me?"
- The presenter is either a fellow teacher, an instructional coach, or an admin who has earned classroom credibility — not an outside tech vendor.
- Goal isn't to convert anyone. Goal is: leave with one thing they'd actually try Monday morning.

---

## Tone & design principles (steer every slide by these)

- **Lead with the teacher, not the tool.** AI is the assistant; the teacher is the expert.
- **Talk like a colleague at lunch**, not a keynote speaker. Contractions. Real classroom examples. Named subjects, named grades.
- **Acknowledge the elephant first.** Don't pretend concerns aren't in the room. Naming them = trust.
- **Show, don't pitch.** One real before/after beats five bullet points of features.
- **Warm visual language:**
  - Palette: muted, paper-like. Cream/off-white background. A grounded color (forest green, terracotta, navy, or warm gold) as accent. Avoid Silicon-Valley purple gradients, neon, dark mode tech aesthetic.
  - Typography: a humanist serif for headlines (Source Serif, Lora, Fraunces) + a friendly sans for body (Inter, Source Sans). No monospace. No "futuristic" fonts.
  - Imagery: photos of classrooms, student work, handwritten notes, coffee cups, sticky notes. NOT: robots, brains made of circuits, glowing chips, the word "FUTURE" in all caps.
  - Lots of white space. One idea per slide. Treat it like a picture book, not a dashboard.

---

## Narrative arc

1. **I see you** → Acknowledge the workload reality. Build the room.
2. **Here's what AI actually is (for us)** → Demystify in plain English. Lower the stakes.
3. **Here's where it actually helps** → Concrete lesson-planning use cases.
4. **Watch this** → A real before/after example from one lesson.
5. **Here's where to be careful** → Privacy, accuracy, your voice. Earn trust.
6. **Try one thing this week** → A tiny, low-risk first step + a sentence of permission.

The arc is: *empathy → demystify → utility → proof → guardrails → invitation.* Skip any step and the talk feels either preachy or salesy.

## Slide-by-slide

### Slide 1 — "Last Sunday night."

**Big idea:** Start in the teacher's actual life, not in tech.

**Layout sketch (ASCII):**

```
+----------------------------------------------+
|                                              |
|   Last Sunday night, you were planning       |
|   next week's lessons.                       |
|                                              |
|   It was 9:47 PM.                            |
|                                              |
|                                              |
|   [small warm photo: kitchen table, mug,     |
|    open binder, laptop — softly lit]         |
|                                              |
|                                              |
|   What if you'd had a thinking partner       |
|   who never gets tired?                      |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- Open with the universal moment. No "Hello, today we'll be discussing artificial intelligence." That's the cold tech vibe.
- Pause after "9:47 PM." Let the room laugh / sigh.
- Land the closing line softly. We're not selling. We're naming a possibility.

**Why it works:** Establishes that this talk is *for* them, *about* them. Buys 5 minutes of attention.

---

### Slide 2 — "What this actually is. (And isn't.)"

**Big idea:** Demystify in one breath. Cut the mystique.

**Layout sketch:**

```
+----------------------------------------------+
|  What this actually is.                      |
|  ---------------------------                 |
|                                              |
|   Think of it as a very fast, very patient   |
|   brainstorming partner who has read         |
|   a lot — but doesn't know your kids.        |
|                                              |
|                                              |
|   It's good at:    It's not:                 |
|   • drafts         • a source of truth       |
|   • options        • a replacement for you   |
|   • rewording      • aware of your standards |
|   • summarizing    • aware of your students  |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- The "very patient brainstorming partner" framing is the whole talk in one sentence. Repeat it later.
- The right column matters as much as the left. Saying what it *isn't* is what builds trust.
- Avoid the words "LLM," "model," "GPT," "prompt engineering." You can use "tool" or "assistant."

**Why it works:** Levels the room. The skeptic and the curious teacher now both have the same mental model.

---

### Slide 3 — "Six places it actually helps with planning."

**Big idea:** Concrete utility, not abstract promise. All six are things teachers already do — AI just compresses time on each one.

**Layout sketch:**

```
+----------------------------------------------+
|  Six places it earns its keep                |
|  -----------------------------               |
|                                              |
|  1. Differentiating one worksheet            |
|     into three reading levels                |
|                                              |
|  2. Drafting a hook / opener                 |
|     for a topic you've taught a hundred times|
|                                              |
|  3. Turning a standard into                  |
|     "I can…" statements for the board        |
|                                              |
|  4. Generating discussion questions          |
|     at different DOK levels                  |
|                                              |
|  5. Rewriting a parent email                 |
|     when you're out of patience              |
|                                              |
|  6. Building a sub plan in 10 minutes        |
|     when you wake up sick                    |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- Read all six out loud. Watch which one lands — that's the use case you expand on in Q&A.
- #5 and #6 always get a knowing laugh. Keep them.
- Frame each as "you already do this — what if it took 8 minutes instead of 40?"

**Why it works:** Every item is a thing every teacher recognizes. None of them are "let AI teach your class." The line is clear: AI drafts, teacher decides.

### Slide 4 — "Watch this. (A 5th-grade fractions lesson.)"

**Big idea:** One concrete before/after. This is the slide that converts skeptics.

**Layout sketch:**

```
+----------------------------------------------+
|  A 5th-grade fractions warm-up               |
|  --------------------------------            |
|                                              |
|   BEFORE                  AFTER (8 min later)|
|   ------                  ------------       |
|                                              |
|   "Solve: 3/4 + 1/6"      Same problem,      |
|                           reframed:          |
|                                              |
|                           "Maya is baking    |
|                            cornbread with    |
|                            her grandma.      |
|                            The recipe needs  |
|                            3/4 cup of milk,  |
|                            but they only     |
|                            poured in 1/6.    |
|                            How much more?"   |
|                                              |
|                           + 2 extension Qs   |
|                           + 1 visual model   |
|                           + ELL version      |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- Walk through the *exact* request you made (in human English, not "prompt engineering"). Something like: *"I asked it: rewrite this problem as a short story a 10-year-old would care about, add two harder follow-ups, and a version with simpler English for my newcomers."*
- Critical: say out loud what you *changed* before using it. ("I swapped 'grandma' for 'auntie' because that fit my kids better. It got 'cornbread' from somewhere — fine. It suggested a visual that was wrong, so I tossed it.")
- That edit step is what models the trustworthy workflow. **You stay the teacher.**

**Why it works:** Specificity is credibility. One real example > ten hypothetical features. And the editing moment teaches the actual skill: AI drafts, you decide.

### Slide 5 — "Where to be careful."

**Big idea:** Earn trust by naming the real risks plainly. Don't soft-pedal.

**Layout sketch:**

```
+----------------------------------------------+
|  Where to be careful                         |
|  -------------------                         |
|                                              |
|   Student data                               |
|   Never paste a student's name, IEP, grades, |
|   or anything identifying into a public      |
|   tool. Use initials or "Student A."         |
|   (Check your district's approved tools.)    |
|                                              |
|   Accuracy                                   |
|   It will sound confident and be wrong.      |
|   Treat every fact like a student's claim    |
|   you'd verify before grading.               |
|                                              |
|   Your voice                                 |
|   If a parent email sounds like a robot      |
|   wrote it, a robot wrote it. Edit until     |
|   it sounds like you.                        |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- This slide is the trust transaction. Move slowly through it.
- On "student data": be specific about your district's policy if you know it. If not, say "ask before you paste."
- On "accuracy": share a real moment it lied to you. Vulnerability builds credibility.
- On "your voice": this is what most teachers are actually worried about. Naming it directly is a relief for the room.

**Why it works:** A presenter who says "here are three reasons to be careful" is trusted more than one who says "it's amazing!" Skeptics relax. Curious teachers feel responsible-adult permission to try.

---

### Slide 6 — "One small thing, this week."

**Big idea:** Lower the activation energy to almost zero. End on warmth, not a CTA.

**Layout sketch:**

```
+----------------------------------------------+
|                                              |
|   One small thing, this week:                |
|                                              |
|   Pick the lesson you're least excited       |
|   to teach.                                  |
|                                              |
|   Ask it for three ways to make the          |
|   opening five minutes more interesting.     |
|                                              |
|   Keep what fits you. Throw out the rest.    |
|                                              |
|                                              |
|   ---                                        |
|                                              |
|   You're still the teacher.                  |
|   This just gives you back Sunday night.     |
|                                              |
|   [presenter name + a way to reach them]     |
|                                              |
+----------------------------------------------+
```

**Speaker notes:**
- The assignment is deliberately tiny: one lesson, one opening, five minutes. Anyone in the room can do this.
- Closing line ("This just gives you back Sunday night") closes the loop back to slide 1. Use it.
- Offer to send a one-page handout. Don't promise an "AI training." Trainings feel like work; a handout feels like a gift.

**Why it works:** Ends on agency and on warmth. The teacher leaves with one concrete action and the feeling that the presenter respects them.

## Optional 7th slide (only if there's time / Q&A buffer)

A single-slide "starter kit" with:
- 3 example requests they can paste in tonight (worksheet differentiation, hook brainstorm, sub plan skeleton).
- The district's currently-approved tool, if there is one.
- A name + email of someone (the presenter, hopefully) willing to look at their first attempt.

Put this in a handout instead of a slide if possible. Handouts get kept; slides get forgotten.

---

## What to cut if time gets tight

- Drop slide 2 first. The "what it is / isn't" content can live in slide 3's intro line.
- Never drop slide 4 (the example) or slide 5 (the guardrails). Those are the talk.
- Slide 1 is non-negotiable — it's the only thing that buys the room.

---

## Visual style cheat sheet for whoever builds the deck

- Background: warm off-white (#FBF7F2-ish), or a single dark grounded color (deep forest, navy) with cream text. Never pure white, never pure black.
- One accent color, used sparingly. Terracotta, warm gold, or sage.
- Headlines in a humanist serif. Body in a friendly sans-serif. No display fonts trying to look "innovative."
- Photos must include people, not screens. A teacher's hand writing on a whiteboard > a stock photo of a glowing AI brain.
- No icons of robots. No icons of brains. A small icon of a coffee cup, a sticky note, a notebook? Yes.
- Slide numbers small and quiet. Footer with the date + school/district feels like a memo from a colleague — that's the vibe.

---

## One-line summary

The talk's job isn't to teach AI. It's to make a tired teacher feel respected, briefly curious, and safe enough to try one small thing on Tuesday.
